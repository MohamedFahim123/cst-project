Introduction

TecNo E-Commerce Platform is a modern, responsive web application built with vanilla JavaScript, Bootstrap, and a clean component-based architecture. This project features a multi-role system supporting customers, sellers, and administrators with comprehensive dashboards and functionality.

This documentation provides a complete guide to setting up, using, and developing TecNo E-Commerce Platform. Whether you're an end-user, administrator, or developer, you'll find the information you need to effectively work with this application.

Key Features

Multi-Role System

Support for customers, sellers, and administrators with tailored dashboards and functionality for each role.

E-Commerce Functionality

Product catalog, shopping cart, wishlist, checkout process, and order management system.

Responsive Design

Mobile-first design optimized for all device sizes with Bootstrap integration.

Modern Architecture

SPA architecture with client-side routing, component system, and modern JavaScript features.

Usage Guide

For Customers

  1. Browse Products: Visit the shop page to view available products with filtering options.
  2. Search & Filter: Use the search bar and filters to find specific items.
  3. Add to Cart: Click "Add to Cart" on product pages to add items to your shopping cart.
  4. Manage Wishlist: Save products for later viewing by adding them to your wishlist.
  5. Checkout: Complete your purchase through the secure checkout process.
  6. View Orders: Track your order history in the customer dashboard.

For Sellers

  1. Register as Seller: Create a seller account during registration.
  2. Manage Products: Add, edit, and delete your products from the seller dashboard.
  3. View Orders: Monitor orders for your products and update order status.
  4. Track Sales: View sales analytics and performance metrics.
  5. Update Profile: Manage your seller information and store details.

Deployment Guide

Publishing your application to Vercel

The project is configured for deployment on Vercel with the following configuration in vercel.json:

{
"rewrites": [
{
"source": "/(.*)",
"destination": "/index.html"
},
{
"source": "/robots.txt",
"destination": "/api/robots.txt"
}
],
"headers": [
{
"source": "/robots.txt",
"headers": [
{
"key": "Content-Type",
"value": "text/plain"
}
]
}
]
}

Deploy to Vercel

Using Git Integration:

  1. Push your code to GitHub
  2. Connect your repository to Vercel
  3. Vercel will automatically deploy on every push

Manual Deployment

Upload all files to your web server maintaining the directory structure. Ensure your server is configured to redirect all requests to index.html for proper SPA functionality.

Troubleshooting

Common issues and solutions

Favicon Not Appearing

If the SVG favicon doesn't appear in the browser tab:

  1. Clear browser cache: Browsers aggressively cache favicons
  2. Check file path: Ensure the path /assets/logo.svg is correct
  3. Verify SVG validity: Open the SVG directly in the browser to check if it renders
  4. Add PNG fallback: Some browsers have limited SVG favicon support

Routing Issues

If pages don't load correctly in production:

  1. Ensure your server is configured to redirect all requests to index.html
  2. Verify that the Vercel configuration is properly set up
  3. Check for JavaScript errors in the browser console

Storage Issues

If data isn't persisting between sessions:

  1. Check if the browser allows localStorage and IndexedDB
  2. Verify that storage isn't being cleared automatically
  3. Test in incognito mode with different settings

Performance Optimization

For better performance:

  1. Enable compression on your server
  2. Optimize images before uploading
  3. Consider implementing lazy loading for images
  4. Minify CSS and JavaScript files for production
